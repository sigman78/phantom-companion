# Task 01 - Project Scaffold

## Goal

Bootstrap the Astro + Svelte + TypeScript project with correct config, directory structure,
and a working public symlink to the game data folder.

## Depends on

None. This is the first task.

| Path | Role |
|---|---|
| `astro.config.mjs` | Svelte integration + Vite path aliases |
| `tsconfig.json` | TypeScript path aliases matching Vite |

## Implementation

### 1. Scaffold the project

Run from an empty directory (or the project root if starting fresh):

```bash
npm create astro@latest . -- --template minimal --typescript strict
npx astro add svelte
npm install
```

This generates `astro.config.mjs`, `tsconfig.json`, `package.json`, and `src/pages/index.astro`.

### 2. Configure astro.config.mjs

Replace the generated file with:

```js
import { defineConfig } from 'astro/config';
import svelte from '@astrojs/svelte';

export default defineConfig({
  integrations: [svelte()],
  vite: {
    resolve: {
      alias: {
        '$data': '/public/game-data/data',
        '$art':  '/public/game-data/art',
      }
    }
  }
});
```

### 3. Configure tsconfig.json

Add path aliases under `compilerOptions`. Merge into the generated file (do not replace
the entire file - preserve `"extends"` and `"include"` that Astro generates):

```json
{
  "compilerOptions": {
    "paths": {
      "$data/*": ["./public/game-data/data/*"],
      "$art/*":  ["./public/game-data/art/*"]
    }
  }
}
```

The generated Astro tsconfig already sets `"strict": true` via its base config. Verify
`"moduleResolution"` is `"bundler"` or `"node16"` (Astro's default is fine).

### 4. Create the public symlink

The `data/` folder at project root contains all game assets (115 MB). A symlink in
`public/` makes it accessible at runtime URLs without copying.

**Windows (run once, developer mode or admin):**

```cmd
cd public
mklink /J game-data ..\data
```

Use junction (`/J`) rather than directory symlink (`/D`) to avoid needing admin rights.

**Linux/macOS:**

```bash
cd public && ln -s ../data game-data
```

After creation, verify the symlink works by opening a browser devtools Network tab while
the dev server runs and fetching:
`http://localhost:4321/game-data/data/adversary-stats.json`

The response must be a JSON object with adversary names as keys.

Note: `adversary-stats.json` lives at `data/data/adversary-stats.json` (double `data/`),
so the URL is `/game-data/data/adversary-stats.json`, not `/game-data/adversary-stats.json`.
